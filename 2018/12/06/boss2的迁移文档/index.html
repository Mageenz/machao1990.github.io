boss2的迁移文档（webpack3-4）

发表于 2018-12-06

我坚持认为开发者应该更关注于开发本身，而不是配置项目构建性能等琐事上面。

Vue CLI 3

升级/安装Vue CLI 3

如果已经全局安装了旧版本的 vue-cli (1.x 或 2.x)，需要先通过 npm uninstall vue-cli -g 或 yarn global remove vue-cli 卸载它。

然后npm install -g @vue/cli安装。

创建项目

安装好cli工具后，通过vue ui启动可视化界面，或者简单的vue create boss2-webpack4创建项目。

迁移boss2

拷贝源码

新建配置文件

成果

dll模式已经被去除了。

不同配置的电脑时间应该会有偏差，这是基于我的电脑（i7，16g内存，固态硬盘）。

第一次构建（无cache）：
build：42s（原来65ms）
serve：42s（原来47ms）

之后构建：
build：15s（同上）
serve：15s（同上）

对比原有的构建速度，应该有35%左右的提升（cache后更高达78%）。具体原因较难考究，从表面来看，主要原因应该是性能更好的webpack4，以及借助cache-loader以及thread-loader，以及vue-cli给我们内设的良好配置吧。
